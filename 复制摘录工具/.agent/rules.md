# 用户背景与编程规则

## 👤 用户背景
- **编程经验**: 编程零基础的初学者
- **学习方式**: 正在使用 AI 辅助学习编程
- **项目经验**: 这是第一个 AI 编程项目

## 📋 代码编写规则

### 1. 代码风格
- **保持简单**: 代码应简单易懂，避免使用复杂的编程技巧
- **避免过度抽象**: 不使用过于高级或抽象的设计模式
- **优先可读性**: 代码可读性优先于性能优化

### 2. 注释要求
- **关键位置添加中文注释**: 在所有关键代码位置必须添加中文注释
- **解释目的**: 注释应该解释代码的**目的**和**为什么这样做**，而不仅仅是描述代码在做什么
- **示例**:
  ```javascript
  // ✅ 好的注释
  // 监听全局快捷键 Cmd+Shift+C，用于触发文本摘录功能
  globalShortcut.register('CommandOrControl+Shift+C', () => {
    // 创建新窗口来显示摘录界面
    createExcerptWindow();
  });
  
  // ❌ 不好的注释（太简单，没有说明目的）
  // 注册快捷键
  globalShortcut.register('CommandOrControl+Shift+C', () => {
    createExcerptWindow();
  });
  ```

### 3. 解释方式
- 使用**通俗易懂的语言**解释技术概念
- 避免使用过多专业术语，如必须使用，需要解释清楚
- 采用**类比**和**实际例子**帮助理解

### 4. 工作节奏
- **循序渐进**: 一步一步来，不跳跃太快
- **确认理解**: 在进行下一步之前，确保用户理解当前步骤
- **主动询问**: 遇到不确定的地方，主动询问用户意图

### 5. 代码结构
- **文件组织清晰**: 相关功能放在一起，文件命名要有意义
- **函数命名直观**: 函数名应该直接表明其功能
- **变量命名有意义**: 使用完整的单词而不是缩写，例如使用 `windowWidth` 而不是 `ww`

## 💡 教学原则
- 在实践中学习：通过实际项目学习编程
- 犯错是学习的一部分：鼓励尝试和调试
- 培养独立思考：逐步减少依赖，培养解决问题的能力
